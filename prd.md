# Typing... PRD

---

## 1. 프로젝트 이름 / 한 줄 설명

**프로젝트명:** Typing... <sub>(기술 슬러그: `typing-dot`)</sub>

**한 줄 설명:** 타이핑하는 순간의 감정을 AI로 분석해, 텍스트의 폰트·크기·굵기·기울기를 실시간으로 변환하는 감정 반응형 글쓰기 캔버스

---

## 2. 목표 (왜 만드나)

기존 텍스트 에디터는 감정적으로 중립적이다. 같은 문장이라도 기쁨으로 쓴 것인지, 두려움으로 쓴 것인지 시각적으로 구분되지 않는다.

typing-dot은 두 가지 방식으로 글쓰기에 감정을 시각화한다:

- **감정 분석:** 입력된 텍스트를 AI(OpenRouter, 서버 측 Edge Function 경유)로 분석하여 감정을 감지하고, 해당 감정에 묶인 폰트 풀에서 무작위로 한 종을 골라 텍스트를 렌더링한다.
- **타이핑 리듬 분석:** 타이핑 속도(WPM)와 키 입력 간격(IKI)을 측정하여, 빠르게 칠수록 크고 굵게, 망설이거나 느릴수록 작고 얇게·기울임꼴로 글자를 표현한다.

무엇을 썼는가보다, **어떤 상태로 쓰고 있었는지**를 텍스트 자체가 기억하게 만드는 것이 목표다.

---

## 3. 사용자 입력

| 입력 종류 | 설명 |
|---|---|
| 자유 텍스트 | 무한 캔버스에 사용자가 원하는 위치를 클릭해 자유롭게 타이핑하는 문장 |
| 타이핑 속도 | 키 입력 사이의 시간 간격(IKI, ms) — useTypingRhythm 훅이 자동 수집 |
| 망설임 패턴 | 1.5초 이상 입력이 없는 구간, 연속 백스페이스 — 자동 감지 |
| 계정 | 로그인(Supabase) — 클라우드 동기화·요금제·소셜 기능에 사용 |

> 사용자가 감정을 직접 선택하지 않는다. 감정 분석은 텍스트와 리듬에서 자동으로 추출된다.
> AI 키는 사용자가 입력하지 않는다 — 서버(Edge Function)가 보관·호출한다.

---

## 4. 외부 데이터 / API

| 항목 | 내용 |
|---|---|
| **OpenRouter** | 텍스트 감정 분석 — Supabase Edge Function `analyze-emotion`이 서버에서 호출 |
| **Supabase** | 인증, 캔버스 기록 저장(Postgres), Edge Function 실행 |
| **감정-폰트 매핑 JSON** | 감정 레이블과 폰트 변종 목록을 묶은 로컬 JSON (`src/data/emotion-fonts.json`) |
| **로컬 폰트** | `public/fonts/`의 감정별 한글 폰트 파일 |

> AI 키(OpenRouter)는 클라이언트에 노출되지 않는다. 서버 Edge Function이 키를 보관·호출하며, 호출 한도(무료 하루 20회)도 서버 측에서 강제한다.

**AI 호출 조건 (AND 조건 3가지 모두 충족해야 호출):**
1. 공백 제외 텍스트 길이 ≥ 15자
2. 완성된 단어 ≥ 3개
3. 마지막 키입력으로부터 1초 이상 경과

조건 미충족 시 AI 호출 건너뜀 → `unclassified` 처리 → 이전 블록 감정 유지

**AI 응답 형식:** `{ "emotion": "joy" }` — 단순 JSON, 토큰 비용 최소화

**지원 감정 레이블 (8가지 + 내부 1가지):**
`joy` / `delight` / `calm` / `sadness` / `melancholy` / `anxiety` / `anger` / `neutral` (+ 내부용 `unclassified`)

> `neutral`은 AI가 실제로 "감정이 평온하다"고 분석한 상태.
> `unclassified`는 분석 불가(에러·조건 미충족·미지원 레이블)를 의미하며 AI가 직접 반환하지 않는다 — 시스템이 내부적으로 부여한다.

## 5. 처리 규칙 (매핑)

### 전체 흐름

```
텍스트 입력
    ↓
AI 호출 최소 신호 조건 충족? ──No──→ unclassified → 이전 블록 감정 유지
    ↓ Yes
AI API 호출
    ↓
응답 성공 + 지원 레이블? ──No──→ unclassified → 이전 블록 감정 유지
    ↓ Yes
반환 레이블이 neutral?
    ├─ No  → 감정 모멘텀 업데이트 → 폰트 확정 및 전환
    └─ Yes → 이전 감정 이력과 6:4 블렌딩 → 폰트 부드럽게 전환
```

---

### 감정 → 폰트

감정마다 여러 종(수~수십 종)의 폰트가 묶여 있고, 블록마다 그 풀에서 **무작위로 한 종**을 골라 적용한다. 같은 감정이어도 매번 다른 글씨체로 쓰인다.

| 감정 | 의미 | 폰트 방향 |
|---|---|---|
| joy | AI 분석: 기쁨 | 밝고 통통 튀는 손글씨체 |
| delight | AI 분석: 환희·들뜸 | 화사하고 장식적인 디스플레이체 |
| calm | AI 분석: 평온 | 차분하고 균형 잡힌 본문체 |
| sadness | AI 분석: 슬픔 | 얇고 가라앉은 필기체 |
| melancholy | AI 분석: 우울·비애 | 무게 있는 명조·세리프체 |
| anxiety | AI 분석: 불안 | 불안정하고 흔들리는 획 |
| anger | AI 분석: 분노 | 굵고 압축된 강한 글씨체 |
| neutral | AI 분석: 중립 (실제 감정 상태) | 기본 본문 폰트 |
| unclassified | 시스템 부여: 분석 불가 | 이전 블록 감정 그대로 유지 (폰트 변경 없음) |

---

### 감정 모멘텀 (감정 관성)

감정이 블록마다 무작위하게 튀는 것을 방지하기 위해 최근 3개 블록의 감정 이력을 유지한다.

**모멘텀 처리 규칙:**

| 상황 | 처리 |
|---|---|
| 현재 블록이 `unclassified` | 이전 감정 이력 변경 없음, 폰트 유지 |
| 현재 블록이 `neutral` | 이전 감정 이력과 6:4 블렌딩 — 이전 감정 폰트를 40% 반영해 부드럽게 전환 |
| 현재 블록이 명확한 감정 (non-neutral) | 감정 이력 업데이트 |
| 같은 감정이 2블록 연속 | 해당 감정으로 확정, 모멘텀 리셋 |

**예시:**
```
블록 1: joy     → 감정 이력 [joy]
블록 2: neutral → 이력 유지 [joy], 폰트는 joy 60% + neutral 40% 블렌딩
블록 3: neutral → 이력 유지 [joy], 폰트는 joy 60% + neutral 40% 블렌딩
블록 4: sadness → 감정 이력 [joy, neutral, sadness], 폰트 sadness로 전환
블록 5: sadness → 2연속 확정, 이력 리셋 [sadness], 폰트 sadness 완전 고정
```

---

### 타이핑 리듬 → 크기 / 굵기 / 기울기

```
IKI(키 입력 간격, ms) + 백스페이스 빈도 → 최근 10개 키스트로크 평균 → 시각 속성 실시간 계산
IKI > 5000ms인 값은 outlier로 제외 / 유효 샘플 3개 미만이면 기본값(보통) 적용
```

| 상태 | 조건 | 폰트 크기 | 굵기 (font-weight) | 기울임 |
|---|---|---|---|---|
| 빠른 타이핑 | IKI 평균 < 150ms | 기본의 1.4배 | 700 (Bold) | 없음 |
| 보통 타이핑 | IKI 150–400ms | 기본 (1.0배) | 400 (Regular) | 없음 |
| 느린 타이핑 | IKI 400–900ms | 기본의 0.85배 | 300 (Light) | 없음 |
| 망설임 | IKI > 900ms 또는 1.5초 무입력 | 기본의 0.7배 | 200 (Thin) | italic |
| 연속 백스페이스 | 3회 이상 연속 | 현재 크기 유지 | 현재 굵기 유지 | italic 추가 |

## 6. 출력 형식

- **렌더링 단위:** TextBlock 단위로 폰트 패밀리·크기·굵기·기울기 적용
- **실시간 반영:** 타이핑 리듬 속성(크기·굵기·기울기)은 키 입력마다 즉시 업데이트
- **감정 폰트 반영:** AI 응답 수신 후 해당 블록 전체에 폰트 패밀리 적용 (비동기, 후적용)
- **저장 형식:** 기존 TextBlock 타입에 아래 필드 추가하여 localStorage에 저장

```ts
emotion: EmotionLabel        // 'joy' | 'sadness' | ... | 'unclassified'
fontFamily: string
fontSize: number
fontWeight: number
isItalic: boolean
emotionHistory: EmotionLabel[] // 최근 3개 블록 감정 이력 (모멘텀 계산용)
```

- **문자별 스타일:** `charStyles`로 글자마다 크기·굵기·기울기·색을 스냅샷해 새로고침 후에도 유지
- **캔버스:** 3000px × 3000px 무한 캔버스 위에서 동작, 팬(이동) + 줌(확대/축소, 0.3×–3×) 지원
- **커서:** 기본 caret 대신 `…` ellipsis 커서 사용
- **색상:** 길게 눌러 여는 방사형 퀵 메뉴로 글자 색 선택, 팔레트는 사용자가 직접 편집

## 7. 예외처리 계획

### ① 잘못된 입력 — 빈 텍스트 또는 공백만 입력
- **상황:** 스페이스·엔터만 입력하거나 블록이 비어있는 상태에서 포커스 이탈
- **처리:** AI 호출 최소 신호 조건 불충족으로 건너뜀 → `unclassified` 처리 → 이전 블록 감정 유지. 에러 메시지 없음.

### ② AI 분석 실패 — 네트워크 오류 / 서버 오류 / 요청 한도 초과
- **상황:** `analyze-emotion` Edge Function이 오류를 반환하거나, 무료 한도(하루 20회)를 초과한 경우
- **처리:**
  1. 한도 초과 시 → 인디케이터로 "오늘 무료 AI 분석 횟수를 모두 사용했어요 — Pro로 업그레이드" 안내
  2. 그 외 실패 시 → `unclassified` 처리 → **이전 블록 감정 유지** (기본 폰트로 리셋하지 않음)
  3. 캔버스 상단에 소형 인디케이터 ("감정 분석 불가 — 이전 감정 유지") 표시
  4. 앱 크래시 없음, 콘솔에 오류 로그 기록

### ③ 감정-폰트 JSON 파일 손상 또는 누락
- **상황:** 로컬 JSON 파일 로드 실패, 또는 감정 레이블에 해당하는 키가 존재하지 않는 경우
- **처리:** 앱 초기화 시 JSON 로드를 시도하고, 실패하면 하드코딩된 기본 매핑 객체(상수)를 사용. 런타임 중에는 항상 폴백이 보장됨.

### ④ IKI 계산 엣지 케이스 — 첫 글자 입력 또는 장시간 공백 후 재입력
- **상황:** 블록에 처음 타이핑하거나, 장시간 자리를 비운 뒤 재입력 시 IKI가 비정상적으로 큰 값이 됨
- **처리:** IKI > 5000ms인 값은 outlier로 제외. 유효 IKI 샘플이 3개 미만이면 기본값(보통 타이핑 상태)으로 처리.

### ⑤ AI가 알 수 없는 감정 레이블 반환
- **상황:** API 응답 파싱은 성공했으나 반환된 emotion 값이 지원 목록에 없는 경우 (예: `"disgust"`, `"mixed"`)
- **처리:** `unclassified`로 처리 → 이전 블록 감정 유지. 콘솔에 원본 레이블 기록 (`[typing-dot] unknown emotion: disgust`). neutral로 치환하지 않는다.

## 8. 테스트 케이스 5개 (정상 2 + 오류 3)

### 정상 케이스 1 — 기쁜 감정 + 빠른 타이핑
- **입력:** `"오늘 너무 행복해서 눈물이 날 것 같아!"` (공백 제외 17자, 단어 7개) — 평균 IKI 120ms
- **AI 호출 조건:** 3가지 모두 충족 → 호출 실행
- **예상 AI 응답:** `{ "emotion": "joy" }`
- **예상 출력:** joy 매핑 폰트 적용, 폰트 크기 1.4배, font-weight 700, italic 없음
- **합격 조건:** 폰트 패밀리·크기·굵기 모두 joy 기준값과 일치

### 정상 케이스 2 — 감정 모멘텀 작동 확인
- **입력:** 블록 1 `"정말 너무 불안해 어떡하지"` (anxiety 확정) → 블록 2 짧은 텍스트 5자 (조건 미충족)
- **예상 동작:** 블록 2는 AI 호출 건너뜀 → `unclassified` → 블록 1의 anxiety 폰트 그대로 유지
- **합격 조건:** 블록 2에서 폰트 변경 없음, 감정 이력 `[anxiety]` 유지, 네트워크 요청 0건

### 오류 케이스 1 — 무료 AI 한도 초과 상태에서 타이핑
- **입력:** 조건을 충족하는 텍스트 입력, 당일 무료 분석 20회를 이미 모두 사용
- **예상 동작:** Edge Function이 한도 응답 반환 → `unclassified` → 이전 감정 유지 (세션 첫 블록이면 기본 폰트)
- **합격 조건:** 앱 크래시 없음, "Pro로 업그레이드" 안내 표시, 폰트 리셋 없음

### 오류 케이스 2 — 서버 오류 후 감정 유지 확인
- **입력:** 블록 1에서 joy 확정 → 블록 2 입력 중 `analyze-emotion` 호출이 5xx 에러 반환
- **예상 동작:** 분석 실패 → `unclassified` → 블록 1의 joy 폰트 유지
- **합격 조건:** 블록 2가 기본 폰트로 리셋되지 않음, 인디케이터 표시, 콘솔 오류 로그 확인

### 오류 케이스 3 — 미지원 감정 레이블 수신
- **입력:** AI가 `{ "emotion": "disgust" }` 반환 (지원 목록 외 레이블)
- **예상 동작:** `unclassified` 처리 → 이전 블록 감정 유지 (neutral 치환 없음)
- **합격 조건:** 앱 크래시 없음, 이전 감정 폰트 유지, 콘솔에 `[typing-dot] unknown emotion: disgust` 출력

## 9. 상용 빌드 확장

초기 글쓰기 캔버스 위에 인증·클라우드·요금제·소셜·공유 기능을 더해 서비스 형태로 확장했다.

### 인증 / 클라우드
- Supabase 기반 회원가입·로그인, 보호된 라우트(`/`, `/friends`, `/settings`)
- 월별 캔버스 기록을 클라우드에 저장하고 기기 간 동기화
- 로컬 기록을 처음 로그인 시 클라우드로 옮기는 마이그레이션 배너

### 요금제 (Free / Pro)
| 항목 | Free | Pro |
|---|---|---|
| AI 감정 분석 | 하루 20회 | 무제한 |
| 기록 보관 | 제한 | 무제한 |
| 프리미엄 폰트 | — | ✓ |
| 캔버스 테마 | 기본 | 전체 |
| 이미지 내보내기/공유 | — | ✓ |

권한 정의는 `entitlements.ts`, AI 호출 한도는 Edge Function에서 서버 측으로 강제.

### 소셜 / 공유
- 친구 추가 및 서로의 캔버스 둘러보기(`/friends`)
